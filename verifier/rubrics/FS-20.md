# FS-20

## Score Type Decision

- score_type: binary
- rationale: 1=满足，0=不满足，半对算错。

## Judge Instruction

judge 判断回答是否详细说明/列明了约束指定的具体要素（如条款、指标、参数等）。"详细"意味着不是泛泛提及，而是有展开说明。

## Scoring Rubric

| 分值 | 描述 |
|------|------|
| 1 | 约束指定的具体要素均被详细说明/列明 |
| 0 | 指定要素缺失、或仅笼统提及未展开说明 |

## Checklist

- 约束指定的每个要素是否都出现？
- 每个要素是否有展开说明而非一笔带过？

## Edge Cases

1. **要素数量多**：如果约束指定了 5+ 个要素，漏掉任何一个判 0。
2. **隐含表达**：未使用约束原文的措辞但实质上覆盖了同一含义，判 1。
