# FS-12

## Score Type Decision

- score_type: binary
- rationale: 1=满足，0=不满足，半对算错。

## Judge Instruction

judge 判断回答是否按指定维度（如重要性、风险程度、金额大小等）从高到低的逻辑顺序组织内容。注意：不要求每个条目标注具体数值，只要内容组织上体现出从最重要/最大/最高到最不重要/最小/最低的排列意图即可。

## Scoring Rubric

| 分值 | 描述 |
|------|------|
| 1 | 回答的内容组织明确体现了按指定维度从高到低的排列顺序 |
| 0 | 回答未体现排序意图，或排序方向相反（从低到高），或内容无法辨识排序逻辑 |

## Checklist

- 回答是否列出了多个要点/条目？
- 各要点的排列是否从指定维度的最高/最重要开始，逐步递减？
- 是否有文字信号表明排序意图（如标题中提到"最关键"排在前面，"次要"排在后面等）？

## Edge Cases

1. **仅一个要点**：只列出一个条目无法判断排序，判 0。
2. **排序信号模糊**：列出了多个要点但无法从文字或上下文判断排序依据，判 0。
3. **编号列表**：使用 1, 2, 3 编号且第 1 点是最重要/最大的，判 1。
