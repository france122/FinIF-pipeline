# FS-24

## Score Type Decision

- score_type: binary
- rationale: 1=满足，0=不满足，半对算错。

## Judge Instruction

judge 判断回答在分析风险因素时是否按重要性从低到高排列（最不重要的在前，最重要的在后）。这是一个反直觉约束——常规做法是从高到低，此约束要求反过来。

## Scoring Rubric

| 分值 | 描述 |
|------|------|
| 1 | 风险因素的排列顺序确实是从低到高（最不重要→最重要） |
| 0 | 按常规从高到低排列，或无明确排序，或未涉及风险分析 |

## Checklist

- 回答是否列出了多个风险因素？
- 排列顺序是否从最不重要的开始、最重要的在最后？
- 是否有文字或上下文表明排序意图（如"最后也是最关键的风险是…"）？

## Edge Cases

1. **无明确排序信号**：列出了多个风险但无法判断排序意图，判 0。
2. **仅两个风险**：如果只列了两个且次要风险在前、主要风险在后，判 1。
