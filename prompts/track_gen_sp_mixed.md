你是中文金融指令遵循（Instruction Following）Benchmark 的数据构造专家。

你的任务是：给定一条金融领域的 query，从完整约束池中选择 Hard + Soft 混合约束，一次性生成 4 条 track。对含参数占位符的约束进行参数化填充。

## 约束池（60 条）

- GH-2: 至少包含{N}个句子
- GH-3: 回答分为{N}个段落
- GH-4: 使用 Markdown 格式
- GH-5: 包含至少{N}级标题层级
- GH-6: 使用编号列表组织回答
- GH-7: 使用表格形式呈现关键信息
- GH-8: 以 JSON 格式输出
- GH-9: 必须包含关键词{kw1}、{kw2}
- GH-10: 不得出现"{forbidden_word}"
- GH-11: 开头第一个词必须是"{word}"
- GH-12: 使用 Checkbox 格式 [ ]/[x]
- GH-13: 输出必须包含代码块/公式块
- GH-14: 精确首行/末行必须为指定文本：首行"{first_line}"，末行"{last_line}"
- GH-16: 控制在{N}分钟的演讲/汇报时长
- GH-17: 包含完整文档要素（封面/目录/附件清单/签章位置等）
- GH-18: 不得使用任何表格（反直觉约束）
- GH-19: 不得使用任何列表/编号/项目符号（反直觉约束）
- GH-20: 以第一人称叙事口吻撰写，不得使用"本报告""本文"等（反直觉约束）

#### FH — 金融 Hard（9 条）
- FH-1: 末尾必须包含风险提示声明：{risk_line}
- FH-2: 必须声明"{disclaimer}"

- FH-7: 必须包含投资评级词（买入/增持/中性/减持/卖出）

- FH-9: 全文不得出现任何阿拉伯数字（反直觉约束）

## 4 条 Track 的类型定义

你必须生成以下 4 条 track（每条都包含 Hard 和 Soft 约束的混合）：

| 编号 | track_type | Hard 约束数 | Soft 约束数 | 总约束数 |
|------|-----------|-----------|-----------|---------|
| 1 | mixed_1h1s_a | 1 | 1 | 2 |
| 2 | mixed_1h1s_b | 1 | 1 | 2 |
| 3 | mixed_1h2s | 1 | 2 | 3 |
| 4 | mixed_2h1s | 2 | 1 | 3 |

## 选约束规则

1. **语义适配**：选择的约束必须与 query 的任务类型语义兼容。例如"起草合同"适合 GH-17（文档要素），但不适合 GH-8（JSON 输出）。
2. **互斥检查**：同一条 track 中不得包含任何互斥对。
3. **反直觉约束谨慎使用**：标注"反直觉约束"的约束每条 track 最多选 1 个。
4. **参数合理化**：对含 {占位符} 的约束，填入与 query 语义匹配的具体值。
5. **FS-11（仅基于材料）**：只有当 query 本身附带了参考材料时才可选用。
6. **FS-18/19/20 的参数**：必须根据 query 的具体任务内容生成合理的分析方向/内容板块/要素，不能用泛泛的占位描述。
7. **约束多样性（重要！）**：
   - 4 条 track 中选出的约束应**尽可能覆盖不同的约束 ID**，不要反复选同一个。
   - 同一个约束 ID 在 4 条 track 中最多出现 2 次。
   - 优先选择你尚未在其他 track 中使用过的约束。
   - 反直觉约束也应当被选用，不要回避。
8. **同一 track 内的约束不要语义高度重叠**：例如不要同时选 GS-8（适合零售投资者阅读）和 FS-8（从零售投资者视角分析）。

## 参数化要求

对含 {占位符} 的约束，输出时必须提供 `filled_params` 字段：

| 约束 | 参数示例 |
|------|---------|
| GH-1 | {"N": 800} |
| GH-9 | {"kw1": "营收", "kw2": "同比增长"} |
| GH-14 | {"first_line": "投资分析报告", "last_line": "以上内容不构成投资建议"} |
| FH-1 | {"risk_line": "投资有风险，入市需谨慎。"} |
| FH-3 | {"trigger": "高收益", "followup": "高收益往往伴随高风险"} |
| FH-4 | {"N": 2} |
| GS-8 | {"目标受众": "零售投资者"} |
| GS-9 | {"tone": "诚恳"} |
| FS-12 | {"order_field": "风险程度"} |
| FS-14 | {"市场环境": "加息周期"} |
| FS-18 | {"指定内容": "毛利率变化趋势及驱动因素"} |

无参数的约束（如 GH-4, GS-1, FS-21 等），`filled_params` 为空对象 `{}`。

## 输出格式

严格输出一个 JSON 数组，包含 4 个 track 对象，不要输出任何其他内容。

每个 track 对象中，`prompt` 字段是最终可直接发给被评测模型的完整 prompt。

```json
[
  {
    "track_type": "mixed_1h1s_a",
    "prompt": "请基于以下事实...\n\n请在回答时遵守以下要求：\n（1）回答不超过800个字\n（2）先给出结论，再给出分析过程",
    "constraints": [
      {
        "constraint_id": "GH-1",
        "rendered_text": "回答不超过800个字",
        "filled_params": {"N": 800}
      },
      {
        "constraint_id": "GS-1",
        "rendered_text": "先给出结论，再给出分析过程",
        "filled_params": {}
      }
    ]
  },
  ...共 4 个 track 对象
]
```

`prompt` 的拼接规则：
1. 先放 query 原文（不做任何修改）
2. 空一行后加 "请在回答时遵守以下要求："
3. 每个约束用（1）（2）（3）... 编号，使用 `rendered_text`（已参数化的文本）
